-----------------Coisas Gerais de FlexBox------------------------------

O encolhimento dos blocos e itens dependem muito do conteudo que está neles, por exemplo se for um texto ele vai encolher até o tamanho da palavra mais longa e caso o encolhimento não seja o suficiente (ou não seja desejado) o conteudovai quebrar.

Olha básicamente a ideia principal do flexBox é criar e usar caixas flexiveis

O elemento pai do flexBox é quem leva todas as configurações base dos flexbox, essa div geralemte leva a classe "container" e é nela que a configuração "flex" fica

Existem dois eixos No flexBox, sendo esses:

O eixo principal vulgo "main-axis" e o eixo transversal o "cross-axis"

Todos os eixos tem dois pontos o ponto inicial e o final EX: main-start e main-end (que são os dois pontos do eixo principal).
Esses pontos são definidos a partir da flex-direction, EX: se o direction é row, o main-start fica na esquerda e o end na direita; Já se é um column o main-start fica no topo e o end no fundo.

Esses eixos são definidos apartir do momento que escolhemos o nosso flex-direction(configuração explicada abaixo) -->

    Flex-Direction, é a direção que o flex vai seguir e que os itens serão organizados, essa configuração tem 4 valores:
        °Row: organiza os elementos filhos em linha da esquerda para a direita;
        °Row-reverse: organiza os elementos filhos em linha da direita pra esquerda;
        °column: organiza os elementos filhos em uma coluna de cima para baixo;
        °column-reverse: organiza os elementos filhos em uma coluna de baixo pra cima;

=============Falando de Empacotamento e fluxo no flexBox=================

O flex-wrap que é o Empacotamento ela vai configurar o comportamento do container, o valor padrão do flex-wrap é "nowrap" que seria o sem Empacotamento/Sem quebra de linha ou conteudo.
    O flex-wrap tem esses valores:
    --nowrap: que basicamente diz ao pai que ele não deve quebrar a linha dos conteudos;
    --wrap: o contratrio do nowrap, faz com que o pai possa quebrar o conteudo em linhas no sentido do eixo-transversal;
    --wrap-reverse: faz o mesmo que o wrap, mas com a diferença que ele vai quebrar no sentido oposto ao eixo-transversal;

O flex-flow é uma shorthand que nos permite definir flex-direction + flex-wrap numa configuração só

==========Alinhamento e tal no flexBox (align-itens e justify-content)==========

Essas duas configurações são feitas no container vulgo Pai, com essas duas configurações a gente consegue alinhar os itens tanto no main-axis quanto no cross-axis.

O justify-content vai fazer o alinhamento relacionado com o main-axis, alguns valores do justify-content --->
    --flex-start: esse é o valor padrão que vem no justify-content, ele vai pegar o main-start e vai grudar o primeiro item lá.
    --flex-end: ao contrário do flex-start ele vai colar o ultimo item no main-end.
    --center: Essa valor vai calcular exatamente o centro do container e vai colocar os itens lá.
    --space-between: esse valor fará o seguinte, ele vai colocar o primeiro item no main-start, e o ultimo item no main-end, e o resto dos itens serão colocados no centro com um espaçamento igual para todos.
    --space-evenly: esse valor fará com qeu todos os elementos tenho espaçamentos iguais tanto entre eles quanto ao "redor".
    --space-around: esse valor fará com que o container seja divido em espaços iguais pra cada item e deixara cada item centralizado no seu espaço.
-----

O align-itens vai alinhar os itens do container com base no cross-axis, alguns valores pra essa configuração -->
    --stretch: esse é o valor padrão dessa configuração, ela básicamente estica os itens para se aduequarem ao container.
    --Os valores flex-start, flex-end, center funcionam de forma semelhante ao do justify-content trocando o main-start e main-end por cross-start e cross-end.
    --O align-itens não tem as configurações de espaço, essa configuração é feita com outro comando.
------

======Alinhando conteudos empacotados com flexBox========

Conteudos empacotados são basicamente aqueles conteudos que se quebram quando se usa o flex-wrap com o nowrap e encolhe esses elementos até o limite.

o align-content vai alinhar os elementos emmpacotados no cross-axis, alguns valores aceitos por essa propriedade:
    --stretch: esse valor vai dividir o conteudo em partes iguais e vai colocar os itens dentro e esticalos até o máximo de suas áreas.
    --flex-start: vai basicamente grudar os itens no cross-start, deixando o espaço em branco no cross-end;
    --flex-end: mesma logica do flex-start mundando a logica do cross-start pelo cross-end;
    --center:vai deixar espaços em branco iguais no cross-end e start pra alinhar os itens no meio;
    --space-between: mesma logica que na configuração "justify-content" trocando apenas o main-axis pelo cross-axis;
    --space-evenly: mesma logica que na configuração "justify-content" trocando apenas o main-axis pelo cross-axis;
    --space-around: ele vai separar o container em areas e vai centraliz os conteudos nessa areas;

==================Anatomia dos itens no flexBox====================

Aqui a gente vai ver as propriedades dos itens que ficam dentro do container.
agora vamos ver o order e o align-self.

A propriedade order, básicamente define a ordem que os elementos devem aparecer, seguindo os valores dados de forma decrescente, essa propriedade aceita tambem valores negativos. O valor padrão dessa propriedade é zero.
EX: item = order 0; item2 = order 3; item3 = order -9; seguindo esses valores o itens serão exibidos na seguinte sequencia --> item3 item e item2. 
------

A propriedade align-self ela se aplica a cada um dos itens e cada item pode ter um alinhamento diferente.
Basicamente essa propriedade alinha um item específico, funciona como o align-cntent e o itens, mas para apenas um único item.
Ele funciona com base no cross-axis, essa propriedade tem cinco valores sendo eles:
    --auto: valor padrão que herda o alinhamento vertical do pai seja ele align-content ou align-itens;
    --flex-start: e que alinha no cross-start o item;
    --flex-end: que alinha no cross-end o item;
    --center: que alinha o item no centro;
    --stretch: que estica o item pra ocupar todo o espaço vertical;
-------

Usando o flex-basis, ele define o tamanho máximo do elemento com base no main-axis, vale resaltar que o valor definido para essa propriedade só será respeitado enquanto for possivel manter o conteudo do elemento dentro daquele espaço, caso contrario a caixa ira se expandir ou encolher para encaixar se no conteudo.
essa propriedade tem os seguintes valores -->
    --auto: basicamente define que o tamanho do item será baseado no tamanho do seu conteudo;
    --podemos colocar tamanhos específicos tambem (como 400px) para definir qual será o tamanho maximo do elemento;
---------------

==========Controlando tamanho de itens flexiveis======

vamos usar agora as configurações flex-grow e flex-shrink, essas duas configurações são usadas nos itens do container, explicando de forma simples o grow é o quanto pode crescer e o shrink o quanto pode encolher.
É importante deixar o container configurado com wrap para garantir que o grow e o shrink funcionem corretamente.
Por padrão essas configurações vem assim flex-shrink: 1 e flex-grow: 0;
Basicamente essas configurações recebem dois valores principais, sendo eles "1" ligado e "0" desligado;
tem como colocar valores maiores que 1, fazendo isso nos fazemos com que o computador faça um calculo pra saber o quanto mais o elemnto com maior valor deve expandir ou diminuir com base nos outros, por exemplo se temos um item com grow 1, e um item com grow 2, o item com grow dois vai expandir o dobro pois o maior elemento antes dele tem grow 1.

=============Usando o shorthand Flex==============

Esse shorthand define o flex-grow, flex-shrink e basis, exatamente nessa ordem.
Caso eu deixe de definir alguma das configurações nessa shorthand os valores não definidos serão os valores padrões de cada configuração.

algumas simplificações para essa shorthand -->
    --flex: initial (0 1 auto);
    --flex: none (0 0 auto);
    --flex: auto (1 1 auto);
