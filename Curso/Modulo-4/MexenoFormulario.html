A diferença de GET e POST

Pelo que o Guanabara ensinou até agora, a unica diferença entre os dois é que um mostra os dados na URL (GET) e o outro apenas via console/devTolls (POST), isso tem haver com a forma que os dados são enviados.

Quando se usa o GET?

Se usa o GEt quando o que se pede são dados que não são sensíveis (dados sensíveis: senhas, infos bancarias), outro momento bom pra usar é que caso seja uma info que é legal de compartilhar, tipo pesquisas do google, a URL com o metodo GET é compartilhavel. A proposito o GET só aguenta até três mil bytes

Quando se usa o POST?

Pra enviar dados sensíveis junto com o HTTPS, se os dados tiverem mais de 3mil bytes e caso nos precisemos enviar arquivos


Usando os inputs de texto e de senha

Agora vamos usar o input de texto de forma mais correta, porque por exemplo esses dois campos de texto acima podem enviar o form mesmo não tendo sido preenchidos, como resolver esse problema? Simples, usamos o atributo "REQUIRED" no input desejado, a proposito o tipo de input "password" por ser um input de senha, deixa os caracteres digitados no campo anonimos (ou seja fica só aquelas bolinha preta lá.), tem o botão de reset também, que serve pra recarregar o form, nos campos que envolvem textos também podemos definir o tamanho máximo e minimo de caracteres com os atributos "MINLENGHT" e "MAXLENGTH" dentro do input desejado.
Nós podemos tambem aumentar o tamanho das caixas com o atributo SIZE, esse size usa como valor de tamanho o numero de caracteres qeu vao caber ali, poe exemplo se eu der size=10; ele vai ter espaço pra 10 caracteres, o autocomplete no form só tem duas opções "ON" e "OFF" certo? Mas se nos usarmos o autocomplete em inputs podemos definir oq exatamente ele deveria estar sugerindo para o usuario preencher.
